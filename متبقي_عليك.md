# متبقي عليك (خطوات يدوية فقط)

تم إعداد المشروع بالكامل من جهة الكود والإعدادات. المتبقي خطوات تُنفَّذ مرة واحدة منك.

---

## 1. رفع المشروع على GitHub

1. أنشئ مستودعاً جديداً على https://github.com/new (اسم مثل `ruqyah-altatil`).
2. من مجلد المشروع على جهازك:
   ```bash
   git init
   git add .
   git commit -m "Initial: رقية التعطيل جاهز للنشر"
   git branch -M main
   git remote add origin https://github.com/اسم_المستخدم/ruqyah-altatil.git
   git push -u origin main
   ```
   (استبدل `اسم_المستخدم` بحسابك في GitHub.)

---

## 2. بناء iOS ورفع منصة أبل (بدون Mac)

**دليل كامل خطوة بخطوة:** انظر ملف **ارفع_التطبيق_منصة_ابل.md** (من إنشاء Bundle ID و App Store Connect حتى إرسال المراجعة).

ملخص سريع:
1. ادخل إلى https://codemagic.io واختر **Add application**.
2. اختر **GitHub** وربط المستودع الذي رفعته (مثلاً `ruqyah-altatil`).
3. من **Integrations** أضف تكامل **App Store Connect** باسم `codemagic` (مفتاح API من Apple — راجع الدليل).
4. شغّل الـ workflow **"iOS رقية التعطيل"** من لوحة التطبيق.
5. بعد انتهاء البناء: يُرفع البناء تلقائياً إلى **TestFlight**؛ ويمكنك تحميل الـ **.ipa** من Artifacts أيضاً.

---

## 3. رابط معاينة الويب (للمراجعة من الجوال)

- الرابط الحالي: **https://delicate-fairy-e5b2b8.netlify.app**
- إذا توقف: ارفع مجدداً ملف **Ruqyah_للويب.zip** من سطح المكتب إلى https://app.netlify.com/drop لتحصل على رابط جديد.

---

## 4. Google Play (بعد موافقة التحقق من الهوية)

- عند موافقة Google على التحقق من الهوية وتأكيد رقم الهاتف، أكمِل نشر الإصدار من لوحة Play Console كما تم شرحه سابقاً.
- ملف الـ AAB الموقع: من مجلد **رقية_للنشر** على سطح المكتب (أو أعد البناء من المشروع مع `key.properties` ورفع الـ AAB من Codemagic/أداة البناء).
